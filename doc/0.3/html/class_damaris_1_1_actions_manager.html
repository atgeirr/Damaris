Detailed Description

The ActionsManager keeps the list of defined actions It is in charge of loading functions from shared libraries, from scripts, etc. and call there functions upon reception of events.

Adds an action defined through a dynamic library.

Parameters:
[in] eventName : Name of the event taken as a key for this action.
[in] fileName : Name of the dynamic library (.so, .dylib...)
[in] functionName : Name of the function to load in this dynamic library.

The action is defined but the dynamic library is not loaded. It will be loaded by the action the first time the action is called. This way the clients can have an instance of ActionsManager that just acts as a set of actions from which they can retrieve action's IDs.
